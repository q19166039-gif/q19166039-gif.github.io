## 会话互通原理

Hermes 所有会话（微信/CLI终端）都存储在同一个 `~/.hermes/state.db`，天然互通，无需额外配置。

## 微信端 ↔ 电脑端 继续对话

### 微信聊一半 → 电脑上继续

```bash
# 继续最近的会话（最常用）
hermes --continue

# 按会话标题恢复
hermes --resume "小爱音箱"

# 先查看最近会话，按ID恢复
hermes sessions list
hermes --resume <会话ID>
```

### 电脑聊一半 → 微信上继续

在微信里直接发：

```
/resume
```

或指定会话标题：

```
/resume 小爱音箱部署
```

## 注意事项

| 问题 | 说明 |
|------|------|
| ✅ 会话数据互通 | 同一个 state.db，天然互通 |
| ⚠️ 模型可能不同 | 微信端和 CLI 可能配了不同模型，恢复后模型不会自动切换 |
| ⚠️ 上下文保留 | 恢复后历史对话都在，但微信端上下文窗口有限，长会话可能被压缩 |
